# 剧情脚本编写

Ink 脚本的语法请参阅 [官方教程](https://github.com/inkle/ink/blob/master/Documentation/WritingWithInk.md)。

本文档的剩余内容可能会用到 Ink 脚本语言的相关定义。

引擎支持的变量类型：Bool、Int32、Float32、String

## 样式代码

剧情脚本的文本支持使用 [格式化代码 - 中文 Minecraft Wiki](https://zh.minecraft.wiki/w/%E6%A0%BC%E5%BC%8F%E5%8C%96%E4%BB%A3%E7%A0%81) 进行转义。和旧版本 Minecraft 所使用的样式代码不同的是，Inkraft 使用 `&` 作为转义字符而不是 `\u00A7`，同时还支持使用 `&$<六位Hex>` 转义任意 RGB 颜色。

## 引擎函数

本节介绍的是 Inkraft 所提供的引擎函数。



在 Ink 脚本的头部使用 `EXTERNAL <函数定义>` 来声明引入外部函数，Inkraft 所提供的引擎函数分为系统函数和游戏函数。前者是和 Ink 脚本运行有关的函数，后者是和操作 Minecraft 游戏内容有关的函数。

在附录 A 中提供了一个包含全部引擎函数的声明文件，用来提供在编辑器中的语法提示，在脚本编写时可以直接使用 `INCLUDE` 来包含。

在 Inkraft 引擎中，函数名一般使用 camelCase 。

函数参数的“可空”指可以用空字符串 `""` 来占位，因为 Ink 语言不支持函数重载故以此权宜。

#### 引擎变量函数

变量名推荐使用 snake_case ，最好在最前面冠上数据包的命名空间 id 以避免和其他数据包冲突。

如需在玩家间共享的变量，建议通过游戏函数运行服务器命令，辅以原版的计分板或 Storage 机制实现。

| 函数定义                     | 函数说明             | 参数说明                      | 返回值说明                           |
|--------------------------|------------------|---------------------------|---------------------------------|
| hasVariable(name)        | 检查引擎变量是否存在于玩家身上。 | name：变量名。                 | bool，true代表存在，false代表不存在。       |
| getVariable(name)        | 获取引擎变量值。         | name：变量名。                 | any，引擎变量值。<br />变量不存在时返回 false。 |
| setVariable(name, value) | 设置引擎变量值。         | name：变量名。<br />value：变量值。 |                                 |
| unsetVariable(name)      | 擦除引擎变量。          | name：变量名。                 |                                 |
| clearVariables()         | 清空全部引擎变量。        |                           |                                 |

### 函数调用参数字符串转义

在编写 Ink 脚本的过程中经常遇到一些字符需要转义，因此引擎提供了字符串转义功能，仅用于函数的参数中。

所有传递给函数的参数都会经过转义。

使用 `$` 作为转义字符。

| 转义序列 | 对应字符 |
|------|------|
| $$   | $    |
| $<   | {    |
| $>   | }    |
| $q   | "    |
| $d   | '    |
| $s   | #    |
