# 使用方法

## 基本概念

- **引擎**：Inkraft 本身。
- **剧本（剧情脚本）**：引擎执行的最小单元，是由 Ink 脚本语言编写并编译（导出）的 Json 文件。
- **引擎函数**：剧本和游戏交互的桥梁，可以在剧情内容中调用。
- **系统函数**：和控制引擎本身的引擎函数。
- **游戏函数**：和 Minecraft 交互的引擎函数。
- **引擎变量**：由引擎提供的，相对于剧本独立的（可以跨剧本使用的），存储在玩家身上的变量。



### 剧本编写

请参考：[剧情脚本编写](剧情脚本编写.md)



## 添加剧本

将导出的以 `.ink.json` 结尾的 Json 文件放置于数据包的 `data/<命名空间>/inkraft_story` 文件夹下，然后重载数据包。

剧本的命名空间 ID 即 `<命名空间>:<剧本路径>` ，例如如果一个剧本文件在 `data/example/inkraft_story/npc1/talk.ink.json` ，那么它的命名空间 ID 就是 `example:npc1/talk` 。



## 权限

### NeoForge

- `inkraft.use`：使用的权限节点，默认权限等级为 0。
- `inkraft.admin`： 管理员的权限节点，默认权限等级为 2 。

### Fabric

- 使用的权限等级为 0。
- 管理员的权限等级为 2 。



## 命令

Inkraft 引擎在游戏中可以以命令的方式交互。



### 运行

运行一个新的剧本。

命令：`/inkraft start <id> [player]`

id：剧本的命名空间 ID。

player：填一个玩家或者选择出单个玩家的选择器，代表操作的主体，当且仅当有管理员权限时可用。



### 恢复

继续一个正在运行的剧本，或者再次显示最后出现的对话和选项。

命令：`/inkraft current [player]`

player：填一个玩家或者选择出单个玩家的选择器，代表操作的主体，当且仅当有管理员权限时可用。



### 继续

做出选择或者点击继续。

命令：`/inkraft next <token> [optionIndex]` 或 `/inkraft next <player> [optionIndex]`

token：为了防止玩家点击已过期选项生成的临时值，正常情况下不会展示给玩家，也无法获取。

player：填一个玩家或者选择出单个玩家的选择器，代表操作的主体，当且仅当有管理员权限时可用。

optionIndex：选项的序号，从 0 开始，没有可选项的时候不需要填。



### 重置（管理员）

重置引擎状态，会删除包括剧本的运行状态和所有存储的引擎变量的所有信息，当且仅当有管理员权限时可用。

命令：`/inkraft reset [player]`

player：填一个玩家或者选择出单个玩家的选择器，代表操作的主体，当且仅当有管理员权限时可用。



### 获取引擎变量值（管理员）

命令：`/inkraft variables get <name|{<player> <name>}>`

name：引擎变量名，任意字符串。

player：填一个玩家或者选择出单个玩家的选择器，代表操作的主体。



### 设置引擎变量值（管理员）

命令：`/inkraft variables set <name|{<player> <name>}> <value>`

name：引擎变量名，任意字符串。

player：填一个玩家或者选择出单个玩家的选择器，代表操作的主体。

value：引擎变量值，任意字符串，引擎会尝试自动转换为支持的变量类型，如果无法自动转换则会当作字符串处理。



### 查看引擎版本

汇报问题时需要。

命令：`/inkraft version`



## 谓词

Inkraft 提供了战利品表条件 `inkraft:story` 用于判断玩家是否在运行一个剧本。它接受一个可选字符串参数 `story` 输入剧本的命名空间 ID ，留空即对任意剧本都有效。

同时也提供了谓词 `inkraft:in_any_story` 用于判断玩家是否在运行任意剧本。

谓词的编写参见 [谓词 - 中文 Minecraft Wiki](https://zh.minecraft.wiki/w/%E8%B0%93%E8%AF%8D) 。
